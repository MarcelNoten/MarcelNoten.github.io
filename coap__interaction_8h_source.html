74 /*
75  * There are 3 Types of interactions, defined by the Role: CLIENT, SERVER, NOTIFICATION
76  *
77  * CLIENT interactions are created by sending a request to a CoAP server when this lib is used as a client
78  * this usecase is not well tested yet and need further refactoring
79  *
80  * SERVER interactions are created by receiving a request from a CoAP client
81  * and end with the response from the server (this lib) which might be
82  * a single ACK or ACK + CON/NON for postponed responses
83  *
84  * NOTIFICATION interactions are created by receiving a request with the notify option set to 0 (= register)
85  * and can be canceled with a NAK or a new request matching the token with the notify option set to 1 (=unregister)
86  *
87  *
88  */
