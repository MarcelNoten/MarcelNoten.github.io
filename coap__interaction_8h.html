◆ CoAP_InteractionState_t

enum CoAP_InteractionState_t

State of the coap interaction.

Enumerator
COAP_STATE_NOT_SET 

set after memory allocation

COAP_STATE_HANDLE_REQUEST 

Remote request received & parsed -> invoke resource handler [server].

COAP_STATE_RESOURCE_POSTPONE_EMPTY_ACK_SENT 

Wait some time for resource to become ready, meanwhile empty ack has been sent [server].

COAP_STATE_RESPONSE_SENT 

NON or piggy-ACK response has been sent -> wait some time then delete [server].

COAP_STATE_RESPONSE_WAITING_LEISURE 

CON response has been sent (+previous separate ACK of req) [server] -> wait for ACK or resent after some time [server].

COAP_STATE_READY_TO_NOTIFY 

Notification is ready to be send (triggered by resource event) [notificator].

COAP_STATE_NOTIFICATION_SENT 

NON response has been sent -> wait some time then delete [notificator], CON response has been sent -> wait for ACK or resent after some time [notificator].

COAP_STATE_READY_TO_REQUEST 
COAP_STATE_WAITING_RESPONSE 

resend request after some time if no response or ack has been received for some time [client]

COAP_STATE_HANDLE_RESPONSE 

call client callback and delete after some time, ack maybe send too [client]

COAP_STATE_FINISHED 

can be deleted / freed [server]+[notificator]+[client]
